# 🚀 نظام السيستمات الجاهزة — تحويل OPEN.FORM لمنصة عمليات تجارية

---

## 💡 الرؤية الكبيرة

بدلاً من أن يكون المنتج مجرد **"منشئ نماذج"**، يتحول إلى:

> **منصة no-code لتشغيل العمليات التجارية** — كل صاحب مشروع يختار "سيستم" جاهز ويبدأ شغله في دقائق.

الفرق الجوهري:
| نماذج عادية | سيستمات جاهزة |
|---|---|
| سؤال → إجابة | تدفق عمل كامل (Workflow) |
| شخص واحد يتعامل معه | أدوار متعددة (أدمن، موظف، عميل) |
| بيانات ثابتة | حالات متغيرة (Status Flow) |
| لا يوجد متابعة | إشعارات + تقارير + لوحة تحكم |

## 🏗️ الأنظمة المقترحة (بالأولوية)

---

### 1. 🚚 نظام شركات الشحن والتوصيل

**المشكلة التي يحلها:** شركات الشحن الصغيرة تعمل بـ WhatsApp وورق — فوضى كاملة.

**كيف يشتغل:**
```
العميل يطلب شحنة (نموذج عام)
    ↓
الأدمن يرى الطلب في لوحة التحكم
    ↓
يسند الطلب لمندوب معين
    ↓
المندوب يرى طلباته (لوحة تحكم مخصصة)
    ↓
يحدّث الحالة: (استلمت → في الطريق → تم التسليم / مرتجع)
    ↓
العميل يستلم إشعار تلقائي بكل تحديث
    ↓
الأدمن يرى تقرير نهاية اليوم لكل مندوب
```

**الميزات:**
- 🗺️ إسناد طلبات للمناديب
- 📊 لوحة تتبع الشحنات live
- 🧾 فاتورة تلقائية لكل شحنة
- 📱 واجهة مناديب مبسطة (موبايل فرست)
- 📈 تقارير: أكثر مناديب، أكثر مناطق، متوسط وقت التسليم

---

### 2. 🍽️ نظام منيو المطاعم والأوردرات

**المشكلة التي يحلها:** المطاعم الصغيرة تدفع آلاف لأنظمة POS ضخمة.

**كيف يشتغل:**
```
العميل يفتح QR code على الطاولة
    ↓
يشوف المنيو التفاعلي (بالصور والأسعار)
    ↓
يضيف للسلة ويطلب
    ↓
الطلب يظهر فوراً في شاشة المطبخ (Kitchen Display)
    ↓
الكاشير يشوف الإجمالي ويطبع الفاتورة
    ↓
أونر المطعم يشوف التقارير (أكثر أكلة مباعة، ساعات الذروة)
```

**الميزات:**
- 🍕 بناء منيو بالصور والتصنيفات والأسعار
- 🛒 سلة تسوق داخل النموذج
- 👨‍🍳 Kitchen Display Screen (تحديث real-time)
- 💳 حساب الإجمالي + الضريبة تلقائياً
- 📊 تقارير مبيعات يومية/أسبوعية

---

### 3. 🧾 نظام الفواتير والعملاء

**المشكلة:** أصحاب المشاريع الصغيرة يعملون فواتير يدوياً.

**كيف يشتغل:**
```
تسجيل بيانات العميل (CRM بسيط)
    ↓
اختيار المنتجات/الخدمات من قائمة جاهزة
    ↓
توليد فاتورة PDF تلقائياً بالتصميم المخصص
    ↓
إرسال للعميل عبر البريد / واتساب رابط
    ↓
تتبع حالة الفاتورة: (أُرسلت → مشاهدة → مدفوعة / متأخرة)
    ↓
تقرير الإيرادات الشهرية
```

**الميزات:**
- 📄 فاتورة PDF بشعار المنشأة وبياناتها
- 👥 قاعدة بيانات عملاء بسيطة
- 💰 تتبع المدفوعات والمستحقات
- 🔔 تذكير تلقائي للعملاء المتأخرين

---

### 4. 📅 نظام الحجوزات والمواعيد

**المستهدفون:** عيادات، صالونات، مراكز تدريب، مكاتب استشارات.

**كيف يشتغل:**
```
العميل يختار الخدمة والموعد المتاح
    ↓
يملأ بياناته ويؤكد الحجز
    ↓
يستلم تذكرة + رسالة تأكيد
    ↓
الموظف يرى جدول اليوم (أجندة)
    ↓
تذكير تلقائي للعميل قبل الموعد بـ 24 ساعة
```

**الميزات:**
- 📆 تقويم مواعيد تفاعلي
- ⛔ إدارة الأوقات المتاحة وغير المتاحة
- 🔁 منع التعارض في الحجوزات
- 📱 تذكيرات تلقائية (email/SMS)

---

### 5. 👔 نظام الموارد البشرية البسيط (HR Lite)

**المستهدفون:** شركات صغيرة لا تحتاج لنظام HR ضخم.

**يشمل:**
- ✅ تسجيل الحضور والانصراف (QR أو رابط)
- 📝 طلبات الإجازة وعرضها على المدير
- 💰 صرف المرتبات وإرسال قسيمة الراتب
- 📊 تقارير الحضور والغياب
- 📋 تقييم أداء الموظفين (نموذج دوري)

---

### 6. 🛒 نظام الأوردرات للمتاجر الإلكترونية البسيطة

**المستهدفون:** بيع منتجات عبر سوشيال ميديا.

**يشمل:**
- 📦 كتالوج منتجات مع المخزون
- 🛍️ نموذج طلب مع اختيار المنتجات
- 📍 إدارة عناوين التوصيل
- 💳 متابعة حالة الأوردر
- 📊 تقارير المبيعات

## 🏛️ البنية التقنية المقترحة

```
┌─────────────────────────────────────────────┐
│              System Marketplace             │
│  (المنشئ يختار نظاماً جاهزاً)              │
└─────────────────┬───────────────────────────┘
                  │
    ┌─────────────▼──────────────┐
    │     System Engine          │
    │  ┌─────────────────────┐  │
    │  │  Multi-Role Access  │  │  ← أدوار: أدمن، موظف، عميل
    │  │  Status Flow Engine │  │  ← تتبع الحالات
    │  │  Auto-Notifications │  │  ← إشعارات تلقائية
    │  │  Report Generator   │  │  ← تقارير جاهزة
    │  │  PDF Generator      │  │  ← فواتير وتقارير
    │  └─────────────────────┘  │
    └─────────────┬──────────────┘
                  │
    ┌─────────────▼──────────────┐
    │   Existing Form Engine     │  ← ما بنيناه بالفعل
    └────────────────────────────┘
```

### الجداول الجديدة في قاعدة البيانات:

```sql
-- الأنظمة الجاهزة
system_templates (id, name, type, config_schema, created_at)

-- نسخ الأنظمة لكل منشئ
user_systems (id, owner_id, template_id, config, name, created_at)

-- أدوار المستخدمين في النظام
system_members (id, system_id, user_id, role, invited_email, joined_at)

-- سجلات النظام (الطلبات/الشحنات/الفواتير)
system_records (id, system_id, data, status, assigned_to, created_by, created_at)

-- سجل تغييرات الحالة
system_status_log (id, record_id, old_status, new_status, changed_by, note, created_at)
```

## 🔑 المفاهيم الأساسية للتنفيذ

### 1. Multi-Role Access
كل نظام له أدوار مختلفة بصلاحيات مختلفة:
- **Owner/Admin:** يرى كل شيء، يسند، يعدّل
- **Staff/Employee:** يرى ما خصص له فقط، يحدث الحالة
- **Customer/External:** يملأ النماذج، يتابع طلبه

### 2. Status Flow Engine
كل نظام له مسار حالات مخصص:
```
شحن:  [جديد] → [مؤكد] → [مع المندوب] → [تم التسليم / مرتجع]
مطعم: [جديد] → [قيد التحضير] → [جاهز] → [تم التسليم]
فاتورة: [مسودة] → [أُرسلت] → [مشاهدة] → [مدفوعة / متأخرة]
```

### 3. Real-time Updates
باستخدام Supabase Realtime — الكاشير والمطبخ يشوفوا الطلبات فوراً.

### 4. PDF Generation
توليد فواتير وتقارير PDF بشعار المنشأة — باستخدام `@react-pdf/renderer`.
